# Customer Churn Analysis Project

## Overview

This project demonstrates an end-to-end data analytics workflow, covering data extraction, cleaning, exploratory data analysis (EDA), SQL-based business analysis, dashboard development, and reporting.

The objective is to analyze customer behavior, identify key trends, and generate actionable business insights through data visualization and reporting.

---
## Dataset
The dataset contains customer information, including:

- Customer demographics
- Subscription details
- Service usage
- Payment methods
- Customer churn status
- ...

The data was used to explore customer behavior patterns and factors associated with customer retention and churn.

## Project Workflow

### 1. Data Loading
- Imported the dataset using Python.
- Performed initial data inspection and validation.

### 2. Exploratory Data Analysis (EDA)
- Examined data structure and distributions.
- Identified trends, patterns, and potential data quality issues.
- Created visualizations to understand customer behavior.

### 3. Data Cleaning and feature engineering
- Handled missing values.
- Removed duplicates.
- Corrected data types.
- Standardized and prepared data for analysis.

### 4. SQL Analysis
- Imported cleaned data into MySQL.
- Wrote SQL queries to answer business questions.
- Generated metrics related to customer churn, revenue, contracts, and service usage.

### 5. Dashboard Development （still working）
- Built an interactive Tableau dashboard.
- Visualized key performance indicators.
- Enabled filtering and exploration of customer segments.

### 6. Reporting & Presentation （still working）
- Summarized findings in a business report.
- Created a presentation deck using Gamma AI.
- Communicated insights and recommendations for decision-makers.

---

## Key Results

Some of the main insights identified include:

- Customers on month-to-month contracts showed higher churn rates.
- Certain payment methods were associated with increased customer turnover.
